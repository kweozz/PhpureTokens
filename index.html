PHPure Design System

Components for Hyvä · Tailwind v4

A reusable, multi-brand component library bound to the PHPure design tokens. Every visual value comes from the token set — switch data-brand on any parent to see the same component re-skin live. Use the toggles in the top bar to preview golf ↔ mini-golf and to constrain a section to a mobile or tablet viewport.

Foundations

Token architecture

Four layers: core primitives → brand/<name> brand identity → semantic/<name> meaning layer → component/<name> component tokens. Components only bind to component/* or semantic/* — never to core or brand directly. Brand switching happens via data-brand overriding the same variable names with different values.

Foundations

Colors

Semantic background, text, and border tokens. Bound to --color-* CSS custom properties and updated automatically when the brand changes.

Components

Component

Button

Three variants — primary, secondary, tertiary — each with default, hover, active, disabled, and focus states. Bound to --button-{variant}-{slot}-{state}.

<button class="btn btn--primary">Shop now</button>
<button class="btn btn--secondary">View bestsellers</button>
<button class="btn btn--tertiary">Learn more</button>
<button class="btn btn--primary" disabled>Disabled</button>
Component

Link

Standard text links and knockout links for dark backgrounds.

Inline links use the primary brand color and follow underline-on-default with hover and visited states.

On a dark surface, use the knockout link variant instead. 
<a href="#" class="link">Standard link</a>
<a href="#" class="link link--knockout">Knockout link</a>
Component

Form

Text input, textarea, select, checkbox. Bound to --form-* with focus ring on --form-border-focus.
